# Candidate Price Snapshot Contract (PR 3 of 6)

## 1. Purpose

This document defines the deterministic, auditable **candidate price
snapshot** layer introduced in **PR 3** of the Polycopy recovery
sequence. PR 3 collects the market and execution inputs that will
later feed the **Trade Copyability Score** in a subsequent PR.

**PR 3 does NOT:**

- Create signals, orders, or positions.
- Approve paper orders or place live trades.
- Calculate a final Trade Copyability Score.
- Change wallet-scoring formulas, thresholds, or any
  `broker_mode` / `paper_mode` / `is_live` / `order_kill_switch`
  / timer / Caddy / Hermes / `.env` setting.
- Wire snapshot creation into `scripts/run_scan.py` (the
  placeholder `_generate_signals` is byte-for-byte unchanged).
- Begin PR 4.

PR 3 produces a single append-only audit row per `snapshot_run_id`
per candidate.

## 2. Scope

PR 3 ships:

- Public Polymarket CLOB order-book retrieval
  (`PolymarketClobClient`, no auth, no signing, no wallet
  private key, no `py-clob-client` dependency).
- Normalized bid/ask book parsing with full input validation
  (Decimal parsing, NaN / Inf / negative / > 1 / malformed
  string rejection, crossed-book detection).
- Side-aware executable price (BUY → best_ask; SELL → best_bid).
- Side-aware price deterioration.
- Spread, executable-side depth, deterministic expected fill price.
- Trade age, market-declared end time, seconds-until-market-end.
- Bounded fetch statuses (10 distinct values — see §5).
- Append-only snapshot persistence
  (`candidate_price_snapshots` table).
- Tests, documentation, safe disabled-by-default configuration.

PR 3 does NOT perform depth-weighted fill simulation. That belongs
to a later stage.

## 3. Existing Schema v9 Dependency

PR 3 builds on the **v9 schema bridge** that was merged
separately (`fix/align-production-schema-v9` → PR #15). The v9
migration is purely additive: it adds the empty
`candidate_price_snapshots` table and three indexes
(`idx_cps_candidate_fetched`, `idx_cps_status`,
`idx_cps_run`) and registers them in `MIGRATIONS[9]`.

**PR 3 does NOT modify `src/polycopy/db/schema.py`.** The bridge
PR owns the schema.

The v9 table is append-only, has no TTL or deletion job during
the paper-research period, and has **no pointer column** on
`copy_candidates`. The "latest snapshot" is a **query**, not a
foreign key — there is no circular FK.

```sql
UNIQUE(candidate_id, snapshot_run_id)
```

is the only persistence identity.

## 4. Snapshot Identity

| Field | Value |
| --- | --- |
| `id` | UUIDv4 (generated by the engine; never reused) |
| `candidate_id` | FK to `copy_candidates(id)` |
| `snapshot_run_id` | UUID supplied by the caller of `snapshot_one` |
| `fetched_at` | UTC ISO-8601 of the start of the snapshot |
| `created_at` | UTC ISO-8601 of when the row was inserted |

A retry inside one HTTP request is NOT a separate snapshot
(`request_attempts` is recorded on the single row). Re-running
with the **same** `snapshot_run_id` is **idempotent**
(`INSERT OR IGNORE` on the unique key). A new `snapshot_run_id`
creates a new observation.

## 5. Fetch Statuses

`SnapshotFetchStatus` is a **bounded enum** with exactly ten
values:

| Status | Meaning |
| --- | --- |
| `OK` | Both best bid and best ask exist; executable price exists for the candidate side. |
| `EMPTY_BOOK` | No valid bids and no valid asks. |
| `ONE_SIDED_BOOK` | Only bids or only asks exist, OR the candidate's required executable side is missing. |
| `MISSING_TOKEN` | Candidate or market outcome lacks `clob_token_id`. No CLOB call made. |
| `NOT_PENDING` | Candidate status is not `PENDING_PRICE_CHECK`. No CLOB call made. |
| `MARKET_NOT_OPEN` | Persisted market row is closed, resolved, or inactive at snapshot time. No CLOB call made. |
| `RATE_LIMITED` | CLOB returned HTTP 429. Bounded; not retried beyond `clob_max_retries`. |
| `HTTP_ERROR` | Exhausted 5xx retries without recovery. |
| `TIMEOUT` | Final timeout after retry ceiling. |
| `PARSE_ERROR` | Malformed body, NaN / Inf / negative / > 1 price, negative size, invalid structure, or crossed book. |

**No `PENDING` snapshot status exists.** A snapshot row is
either a completed observation (one of the ten above) or it is
not persisted at all.

## 6. Public CLOB Adapter

`src/polycopy/adapters/polymarket_clob.py` exposes
`PolymarketClobClient`, an injected, dependency-injectable HTTP
client. The engine accepts any `BookProvider` Protocol (see
`polymarket_clob.py`).

| Property | Value |
| --- | --- |
| Endpoint | `GET <base_url>/book?token=<token_id>` |
| Authentication | None. |
| Wallet private key | None. |
| Signing | None. |
| API secret / cookie | None. |
| Library | Stdlib `httpx` (no `py-clob-client`). |

The base URL defaults to `https://clob.polymarket.com` and is
configurable via `POLYCOPY_CLOB_BASE_URL`. A trailing slash is
normalized off at parse time. Non-http(s) schemes are rejected.

`POLYCOPY_CLOB_ENABLED` defaults to `false`; this is the
load-bearing safety invariant — deploying PR 3 must not start
hitting `clob.polymarket.com`.

## 7. Book Normalization

| Field | Source | Validation |
| --- | --- | --- |
| `bids`, `asks` | Parsed from JSON, **not trusted in order** | Reject malformed strings, NaN, Inf, negative prices, prices > 1, negative sizes |
| `best_bid` | Maximum valid bid price | None after per-level validation |
| `best_ask` | Minimum valid ask price | None after per-level validation |
| `best_bid_size` | Sum of sizes at `best_bid` (repeated levels aggregated) | None |
| `best_ask_size` | Sum of sizes at `best_ask` | None |
| `mid_price` | `(best_bid + best_ask) / 2` | None |
| `spread` | `best_ask - best_bid` | None |
| Crossed book (`best_bid > best_ask`) | — | Classified `PARSE_ERROR` |
| Zero-size levels | — | Discarded silently |
| `book_hash` | SHA-256 of canonical sorted-level JSON | Stable for the same levels; changes when any level changes; does NOT include URL / headers / credentials |

The canonical book payload used for hashing is intentionally
closed-form: only the sorted, normalized levels. No URL, no
headers, no credentials ever enter the hash.

## 8. Side-Aware Execution

**The midpoint is NOT an executable fill.** It is stored as a
descriptive market reference for the audit.

For a **BUY** candidate:

| Field | Value |
| --- | --- |
| `executable_price` | `best_ask` |
| `expected_fill_price` | `best_ask` |
| `executable_side_depth` | `best_ask_size` |
| `price_deterioration` | `best_ask - source_trade_price` |

For a **SELL** candidate:

| Field | Value |
| --- | --- |
| `executable_price` | `best_bid` |
| `expected_fill_price` | `best_bid` |
| `executable_side_depth` | `best_bid_size` |
| `price_deterioration` | `source_trade_price - best_bid` |

`executable_side_depth` is **not** `best_bid_size + best_ask_size`.
A buyer cannot consume bid liquidity; a seller cannot consume
ask liquidity.

Common (side-independent) calculations:

- `mid_price = (best_bid + best_ask) / 2`
- `spread = best_ask - best_bid`
- `price_deterioration_pct = price_deterioration / source_trade_price`
- `mid_change = mid_price - source_trade_price`
- `mid_change_pct = mid_change / source_trade_price`

Percentages are **not capped** in storage. Any capping for
display or scoring belongs in a later formula layer. The
`source_trade_price <= 0` guard is enforced before division;
snapshots with non-positive source prices produce NULL
percentages and `MARKET_NOT_OPEN` is not triggered by it
(it's a separate `INVALID_SOURCE` engine rejection, not a
persisted status).

**Interpretation of `price_deterioration`:**

- Positive: our available copy price is **worse** than the
  source trade.
- Zero: same price.
- Negative: our available copy price is **better**.

## 9. Price Deterioration (Reaffirmed)

| Side | Formula | Sign |
| --- | --- | --- |
| BUY | `executable_price - source_trade_price` | positive = worse |
| SELL | `source_trade_price - executable_price` | positive = worse |

The raw value is stored. The `pct` form is also stored raw.
Display / scoring layers must apply their own bounds.

## 10. Market End Metadata

PR 3 reuses the existing `markets.end_date` column (populated
from Gamma's `endDate` field by the existing market ingestion
adapter). **No new `markets.end_date_iso` column is added.**

The snapshot copies `markets.end_date` into `market_end_at` so
later edits to the market row do not rewrite the historical
observation. The engine computes
`seconds_to_market_end = market_end_at - snapshot_now` when
`market_end_at` is parseable.

Behavior for NULL / blank / malformed `markets.end_date`:

- **No crash.**
- `market_end_at` is set to the raw source value (NULL preserved).
- `seconds_to_market_end` is `NULL`.

**Negative `seconds_to_market_end` values are valid** and
preserved (a market whose declared end is already in the past
at snapshot time). They are **not clamped**.

`market_metadata_fetched_at` captures `markets.fetched_at`
(audit timestamp of the market row itself), **not** the market
end. It is never used as the current time or as the market
end.

## 11. Persistence and Retention

`src/polycopy/db/price_snapshot_persistence.py` exposes:

- `persist_price_snapshot(db, snapshot) -> tuple[str, bool]`
  returning `(snapshot_id, inserted)`.
- `get_latest_price_snapshot(db, candidate_id) -> Optional[PriceSnapshot]`.

| Behavior | Value |
| --- | --- |
| Insert mode | `INSERT OR IGNORE` |
| Unique key | `(candidate_id, snapshot_run_id)` |
| Same candidate + same run | idempotent (`inserted = False`) |
| Same candidate + different run | new observation (`inserted = True`) |
| Latest query | `WHERE candidate_id = ? ORDER BY fetched_at DESC, id DESC LIMIT 1` |
| Old-row mutation | **NEVER** |
| History rewrite | **NEVER** |
| `decision_log` write | **NEVER** |
| Candidate status update | **NEVER** |
| Signal / order / position write | **NEVER** |
| `latest_price_snapshot_id` column on `copy_candidates` | **DOES NOT EXIST** |

Snapshots are **append-only** during the paper-research period.
No TTL, no deletion job.

The full raw book is **not** stored by default. A bounded
JSON summary is stored:

```json
{
  "best_bid": 0.45,
  "best_bid_size": 10.0,
  "best_ask": 0.55,
  "best_ask_size": 10.0,
  "bid_levels": 1,
  "ask_levels": 1,
  "executable_side": "ASK"
}
```

`book_summary_json` and `book_hash` are the persisted audit
artifacts. The full URL (with token query parameter), request
headers, and credentials are **never** persisted anywhere.

## 12. Disabled-by-Default Configuration

Five `POLYCOPY_*` settings are added (env-prefix binding
verified by test):

| Setting | Default | Validator |
| --- | --- | --- |
| `POLYCOPY_CLOB_ENABLED` | `false` | bool (repo convention) |
| `POLYCOPY_CLOB_BASE_URL` | `https://clob.polymarket.com` | must start with `http://` / `https://`; trailing slash normalized off |
| `POLYCOPY_CLOB_TIMEOUT_SECONDS` | `10` | must be > 0 |
| `POLYCOPY_CLOB_MAX_RETRIES` | `3` | must be ≥ 0 |
| `POLYCOPY_CLOB_RPM` | `30` | must be ≥ 0 (Polycopy safety ceiling, not the platform's documented limit) |

`POLYCOPY_CLOB_ENABLED` is the **runtime gate**; it is a
configuration value with no side effects. Setting it to `true`
does not itself cause an HTTP request. The adapter is
**explicitly instantiated by tests** with a mocked transport;
no production code path instantiates it without an explicit
factory call from the caller.

`POLYCOPY_CLOB_RPM = 30` is a **Polycopy safety default** to
bound blast radius during a future scan pass. It is **not** a
description of the Polymarket platform's documented rate
limit.

**No production `.env` change is performed by this PR.**

## 13. Safety Boundaries

PR 3 commits to:

- **No `scripts/run_scan.py` wiring.** A static source guard
  test in `test_p03_clob_config.py` fails the build if anyone
  imports `PolymarketClobClient`, `snapshot_one`, or
  `persist_price_snapshot` from `run_scan.py`.
- `_generate_signals` placeholder unchanged.
- No signals, orders, positions, or `decision_log` rows
  created by the snapshot path.
- No live trading. No real CLOB calls in tests.
- No wallet private key, no signing, no API secret in the
  CLOB adapter.
- No production CLOB calls (the runtime gate is `false` and
  no production code instantiates the client).
- `BidAskProvider` (`src/polycopy/providers/bidask.py`) is
  unchanged: deterministic, in-memory, offline, non-networked.
- `markets.end_date` reused; **no** `markets.end_date_iso`
  column.

## 14. Testing Contract

The PR 3 test suite is offline-only. Zero network access in CI.
All HTTP is mocked through `httpx.MockTransport` (or a similar
in-test handler). A `@pytest.mark.live` marker may be applied
to live CLOB integration tests, but **such tests must skip by
default** in CI.

The suite covers:

- **Config:** defaults, bool-parsing, env-prefix binding,
  trailing-slash normalization, non-http scheme rejection,
  empty-URL rejection, timeout > 0, retries ≥ 0, RPM ≥ 0,
  flag does not itself make a request.
- **Adapter:** GET `/book` path, `token` query parameter
  encoding, no auth / no signing, unsorted level ordering,
  repeated best-price size aggregation, zero-size level
  discard, malformed string rejection, NaN / Inf rejection,
  negative price / size rejection, price > 1 rejection,
  missing keys, non-list values, empty book, bid-only /
  ask-only books, crossed-book detection, deterministic hash,
  hash changes on level change, retry ceiling, 429 handling,
  exhausted 5xx handling, limiter per outbound attempt.
- **Engine:** BUY execution, SELL execution, positive and
  negative deterioration, zero source-price guard, missing
  token skips provider, non-pending skips provider,
  inactive / closed / resolved market skips provider, empty
  and one-sided books, existing `markets.end_date` behavior,
  missing / malformed / past / future end date, one UTC
  `now` value, `market_metadata_fetched_at` from
  `markets.fetched_at`, no action-table writes.
- **Persistence:** same `run_id` idempotent, different
  `run_id` appends, latest `fetched_at` ordering, `id DESC`
  tie-break, no old-row mutation, no candidate status
  mutation, no `decision_log` write, no signals / orders /
  positions writes.
- **Runtime isolation:** `scripts/run_scan.py` does not
  import or call `PolymarketClobClient`, `snapshot_one`, or
  `persist_price_snapshot`. Verified by static source guard.
